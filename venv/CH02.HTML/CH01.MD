# 웹 기초

## HTML, CSS, JS
```buildoutcfg
* HTML (Hyper Text Markup Language)  
    - 페이지의 제목, 문단, 표, 이미지, 동영상 등의 웹의 구조를 담당
* CSS ( Cascading Style Sheets ) 
    - 실제 화면에 표시되는 방법(색상, 크기, 폰트, 레이아수 등)을 지정해 콘텐츠를 꾸며주는 시각적인 표현(정적)을 담당
* JS ( Java Script )
    - 콘텐츠를 바꾸고 움직이는 등 페이지를 동작시키는 동적 처리를 담당.
```

## 웹앱(웹사이트)의 동작 원리
```buildoutcfg
* 요청 -> 응답으로 이루어져 있음
* 예시 
     사용자가 크롬을이용하여 google.com에 '요청'하여 google.com과 연결된 서버가 여러가지 처리를 진행 한 후 '응답'한다.   
```

## 웹표준
```buildoutcfg
* 웹 표준(W3C)
    웹 표준(Web standard)이란 '웹에서 사용되는 표준 기술이나 규칙'을 의미, W3C의 표준화 제정 단계의 '권고안(REC)'에 해당하는 기술.

* 크로스 브라우징
    크로스 브라우징 (Cross Browsing)이란 조금은 다르게 구동되는 여러 브라우저에서, 동일한 사용자 경험(같은 화면, 같은 동작 등)을 줄 수 있도록 제작하는 기술 방법
* 뷰포트
    웹사이트가 출력되는 영역, 렌더링 되는 곳
```

## 요소의 기본관계
```buildoutcfg
<div><div>Contents</div></div>

* 부모요소, 자식요소가 생김 : 바깥쪽 <div> 부모요소, 안쪽 <div> 자식요소 
* 들여쓰기로 구분

1. <div>
    2. <div>
        3. <div>Contents</div>
    </div>
</div>

✔ 3번의 시점에서 봤을 때
2. 부모요소  1. 상위(조상)요소

✔ 1번의 시점에서 봤을 때
2. 자식요소 , 3. 하위(후손)요소

#----------------------------------------

1. <div>
    2. <div>Contents1</div>
    3. <div>Contents2</div>    
</div>

2. , 3. 은 형제요소
1. 입장에서 2.은 다음요소
2. 입장에서 1.은 이전요소
```
## 글자와 상자 
```buildoutcfg
* 요소가 화면에 출력되는 특성, 크게 2가지로 구분.

인라인 요소 : 글자를 만들기 위한 요소들. 수평으로 쌓임
    - 특징
        * 요소들이 수평으로 쌓임 (1)
        * 포함한 콘텐츠 크기만큼 자동으로 줄어든다.
        * 가로,세로 너비를 지정하는 CSS 속성을 넣어도 반응 없음!
        * margin(외부여백), padding(내부여백)은 인라인 요소는 위아래는 지정할 수 없다.
        * 인라인 요소의 자식요소로 블록 요소가 들어갈 수 없다. 인라인 요소 안에 인라인 요소는 가능하다          

(1)         
<span>Hello</span> --> 대표적인 인라인 요소
<span>World</span> 

>>>> 왼쪽에서 오른쪽으로 수평으로 쌓임
>>>> Hello World 

블록(Block) 요소 : 상자를 만들기 위한 요소들. 수직으로 쌓임
    - 특징
        * 요소들이 수직으로 쌓임
        * 부모요소의 크기만큼 가로가 자동으로 눌어남, 세로는 자동으로 줄어듬
        * 가로, 세로 너비 적용가능
        * 마진,패딩 외,내부전부 적용 가능
        * 블럭요소의 자식요소로 블럭요소, 인라인요소 가능
```
